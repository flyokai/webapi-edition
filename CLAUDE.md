# CLAUDE.md

This is **`flyokai/webapi-edition`** — a project template (used via `composer create-project`) for HTTP/JSON web APIs on the Flyokai framework. It does **not** include the data-service socket stack; for that, see [`flyokai/data-service-edition`](https://github.com/flyokai/data-service-edition).

## Project shape after install

```
.                        # this template — cloned via composer create-project
├── composer.json        # frozen dependency set
├── composer.lock        # committed; see README.md "composer.lock is committed"
├── scripts/sync-agent-assets.php
├── README.md, CLAUDE.md, .gitignore
├── vendor/              # populated by composer install
├── bin/                 # generated by `vendor/bin/flyok-setup install` (DeployAssets step)
├── storage/             # generated; holds flyok.config.json, certs, oauth keys, …
├── .agents/{commands,skills}/   # symlinked from vendor/*/.agents/* by sync-agent-assets.php
└── .claude/{commands,skills}/   # same — synced for Claude Code agents
```

## Commands

```bash
# Install (first time)
vendor/bin/flyok-setup install --db-host=… --db-user=… --db-pass=… --db-name=… --base-url=…

# Subsequent
php bin/flyok-setup upgrade
php bin/flyok-console ...
php bin/flyok-cluster {start|stop|restart}
```

## Agent guidance

- For framework architecture, read `vendor/flyokai/flyokai/README.md` and `vendor/flyokai/flyokai/AGENTS.md`.
- For DI internals, read `vendor/flyokai/flyokai/docs/amphp-injector-flyokai.md` and `vendor/flyokai/amphp-injector/README.md`.
- For scaffolding new modules / endpoints / DTOs / commands, use the synced skills in `.claude/skills/` (e.g. `flyokai-new-module.md`, `flyokai-new-endpoint.md`).
- The HTTP server is an AMPHP `SocketHttpServer` — never block the event loop. Use `Amp\delay()` instead of `sleep()`, `async()` for concurrent work.
- The default DB connection pool is `AsyncMysqliConnectionPool`. Switch via the `ConnectionPool` alias in your project's `diconfig.php`.

## Conventions

- All flyokai/* dependencies are pinned to `dev-dev`. The `main` branch is stable; `dev` is active.
- The `composer.lock` is committed. **Do not** silently regenerate it during normal work — only as part of a deliberate "bump dependencies" commit, with a smoke test.
- The `bin/` and `storage/` directories are generated, not authored. Don't commit them.
- The agent-asset symlinks under `.claude/` and `.agents/` are regenerated by `composer run sync-agent-assets`.

## Customising

A new project derived from this template typically:

1. Adds its own package (e.g. `acme/api-module`) under a `vendor/`-namespaced repo and requires it.
2. Registers DI config files via that module's `bootstrap.php` and `Bootstrap\Type\Web`.
3. Tags Solid DTOs with `#[Table]` so the schema framework reconciles MySQL on `setup:upgrade`.

Use the skills in `.agents/skills/` to scaffold each step rather than copy-pasting boilerplate.
